UI Principles in Practice

  • Use Primary (Coral Red) for all key actions and focus indicators.
  • Use Secondary (Teal Blue) for navigation, tab highlights, and status indicators.
  • Use Tertiary (Golden Yellow) sparingly to draw attention to onboarding tips or spotlight areas.
  • Backgrounds should remain light and airy to reflect approachability and simplicity. (Reflected in `body` background color: `#F9FAFB`)
  • Use rounded corners (8–16px) and subtle shadows for a soft, friendly interface. (Reflected in `border-radius: 0.5rem` for many elements and `box-shadow` properties).
  • Animations should be minimal and smooth (0.2–0.3s) for hover, click, and drag events. (Reflected in `transition: all 0.2s ease-in-out` on buttons and color swatches).

Design System Checklist for Developers

Before starting development, a well-defined design system ensures consistency and efficiency. Developers typically look for:

  • Color Palette: Hex codes, usage guidelines (primary, secondary, neutrals, accents).
  • Typography Scale: Font families, weights, sizes, line heights for headings, body text, metadata.
  • Buttons:
    • Primary, Secondary, Tertiary styles.
    • Hover, Active, Focus, Disabled states.
    • Various sizes (small, medium, large).
    • Iconography within buttons.
  • Form Elements:
    • Input fields (text, number, password, email).
    • Placeholder text styling.
    • Error states, success states.
    • Labels, help text.
    • Textareas, Checkboxes, Radio buttons, Select dropdowns.
  • Navigation Components:
    • Navbar, Sidebar, Tabs.
    • Active/inactive states.
  • Cards & Modals:
    • Standard card styles, different variations.
    • Modal/Dialog box structure and styling.
  • Icons: Consistent icon set and guidelines for usage.
  • Spacing & Layout: Defined padding, margin, and grid systems (e.g., 8pt grid).
  • Shadows & Borders: Consistent application of shadows and borders for depth and separation.
